Porquê o Modo Relacional?

De forma a facilitar a gestão de dados relacionados com a gestão de uma exploração agrícola.
A gestão de uma exploração agrícola é feita através de uma base de dados que contém informação sobre os clientes, os seus incidentes e as suas encomendas, sobre os setores, as suas culturas e toda a informação relevante relacionada com estes.

Para isto necessitamos de uma base de dados que nos permita gerir esta informação de forma eficiente e nos permita relacionar os dados de forma a garantir a sua persistência, a sua integridade e a sua confidencialidade, assegurando que o padrão ACID é cumprido. É também importante que seja possível a todo o momento a realização de transações independentes umas das outras, garantido que a informação destas já realizadas não é perdida.

Posto isto, optamos pelo modelo relacional, visto que este nos permite asssegurar tudo acima referido, sendo uma execelente ferramenta quando se trata de gerir informação relacionada.
Embora existissem outras opções, dado que o tipo de dados que pretendemos gerir é relacionado, ou seja, contém várias relações entre si, este modelo foi o mais indicado.

Vantanges do Modo Relacional

  • Tecnologia madura e com bom suporte documental
  • A cargo de empresas bem estabelecidas no mercado
  • Standards SQL estão bem definidos e aceites globalmente
  • Todos os SGBDR garatem transações ACID (Atomicidade, Cosistência, Isolamento, Durabilidade)
